§ 01 THE IDEAS WE BUILT AROUND / principles
01 — the thesis
democratize the skill

AI-assisted code reading is a skill. codewiki hands it to everyone.

Senior engineers run git log --oneline -- path/, trace imports, check Issues, grep for error handlers. They ask AI the right questions in the right order and get deep understanding. Juniors ask "what does this file do?" and get shallow answers. Same AI, wildly different results.

codewiki encodes senior-level investigation — ctags, grep, git log, gh issue view, dependency graphs — into the skill itself. Whoever runs /codewiki gets the analysis a senior would produce after hours of digging. One command. No token waste across the team asking the same questions.

02 — rebuild, don't patch
zero drift

Docs are generated, not maintained.

Incremental doc updates are where LLMs fail hardest — renames leave ghost references, one section drifts while another is fixed, and nobody notices until someone onboards. codewiki throws the previous wiki away every run. Cheaper to regenerate than to reconcile.

03 — the author answers the question
same model

No chatbot bolted on. The writer is the reader.

The same Claude Code that read your repo and wrote the wiki is the one you talk to when you right-click a paragraph. No second model, no API middleman, no "retrieval-augmented hallucination layer." It remembers your repo because it just read it.

04 — your code never leaves
local, offline-capable

A wiki, on localhost. Forever.

No cloud, no "trust us with your source," no auth. .codewiki/index.html is a single file. Commit it to gitignore, serve it on 8080, open it on a plane. Your proprietary code is exactly as private as it was before you ran the command.

§ 02 HOW IT WORKS / pipeline

Four moves. No config files.

Each step happens inside Claude Code — the same process, the same context window. No subprocess calls, no vector DB to stand up, no background daemon eating your RAM.

// the output of step 4 is the input to every future right-click.

01 ⟶
Scan Glob
Walks the tree, detects language(s), picks up package manifests and infra files. Ignores build output and node_modules by default.
~2s
02 ⟶
Read deep Read
Opens the files that actually matter — entrypoints, routes, schemas, core modules — and reads them cover-to-cover.
~10s
03 ⟶
Analyze native AI
Produces one structured component analysis per major directory: purpose, data flow, key types, gotchas. This is the meat of the wiki.
~60s
04 ⟶
Emit & serve Write + Bash
Writes .codewiki/index.html, starts a local server on :8080, and opens your browser. Right-click is live.
~3s
